Detailed Description

Class which defines simulated puck behavior. Used by both the simulated detection/physics engine, and by the strategy engine for predicted where puck will be, in order to decide how to respond.

Static method to update puck state. Note the assumptions currently in place for the physical model:

  • Collision between puck and mallet do not affect the velocity/position of the mallet.

Reference (billiard collisions i.e. 1st year physics): http://www.real-world-physics-problems.com/physics-of-billiards.html

Note that this method DOES modify the passed puck object velocity and position based on wall reflections.
